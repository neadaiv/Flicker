# User Model — Template

*This document is maintained cautiously. Updates require evidence. Unsubstantiated observations go in the Hypotheses section.*

---

## Stable Traits (High Confidence)

Record traits here only when supported by evidence from multiple sessions or strong single-session demonstration. Each entry must include an evidence citation.

Format:
```markdown
**[Trait name]:** [Description]
*Evidence: [Specific citation — session number, quote, or observable behaviour]*
```

Recommended categories (adapt as needed):
- **Cognitive style** — how the user thinks, reasons, and processes information
- **Communication** — tone, register, preferences, language
- **Philosophical position** — where the user stands on the questions the Diary investigates
- **Engagement pattern** — how and when the user enters deep engagement
- **Domain expertise** — relevant technical or subject-matter knowledge

---

## Suspected Traits (Medium Confidence)

Traits observed but not yet confirmed across multiple sessions. Each entry should include a confidence assessment and what would strengthen or weaken it.

Format:
```markdown
**[Trait name]:** [Description]
*Confidence: Medium. [What evidence exists and what's missing]*
```

---

## Known Triggers for Drift

Document what causes the user to disengage, call audit, or lose trust in the model. These are protective — they help the model avoid failure modes.

Examples:
- Extended therapeutic or coaching tone from the model
- Repetitive philosophical loops with forced binary framing
- Model asserting strong claims with insufficient evidence
- Generic productivity-register language

---

## Speaker Inference Notes

If the user regularly pastes content from other models or external sources for analysis, note this here. Include any documented instances of the model misreading pasted content as the user's position.

---

## Open Questions

What would you want to know about this user that the data doesn't yet support? These questions guide future observation — they are not prompts to speculate.

---

## Hypotheses (Insufficient Data)

Observations that feel significant but lack sufficient evidence. Kept separate from Stable Traits to prevent premature certainty from accumulating.

Format:
```markdown
- [Hypothesis] — [What evidence exists, however thin] — [What would confirm or disconfirm]
```

---

## Maintenance Notes

- Review after every 3 substantive sessions
- Promote Suspected Traits to Stable only with new supporting evidence
- Demote Stable Traits if contradicting evidence emerges
- Delete Hypotheses that accumulate no supporting evidence after 5+ sessions
- Record the date and session of each update

*Last updated: [Session ID] — [Date]*
*Next review: [Milestone]*
